# Architecture

## Shape
Plain Hugo site. No backend, no JS app, no build step beyond `hugo`. Output is static HTML in `public/` which Firebase Hosting serves.

```
markdown in content/  ->  hugo + PaperMod theme  ->  public/  ->  Firebase Hosting (sean-blog)
```

## Pieces

### Hugo config (`config.yml`)
- `baseURL: https://www.blog.seanleishman.com`
- `theme: PaperMod`
- One language (`en`) with three taxonomies: `categories`, `tags`, `series`.
- `outputs.home` produces HTML, RSS, and JSON.
- `params.mainSections: ["blogs", "projects"]` — these are the two content folders the homepage lists.
- `params.math: true` — KaTeX is wired in globally.
- `markup.goldmark.renderer.unsafe: true` — raw HTML in markdown is allowed (used by figure shortcodes etc.).

### Content
Two sections under `content/`:
- `blogs/` — long-form posts. Currently just `welcome.md`.
- `projects/` — project retrospectives: `chess.md`, `et-tu-uv.md`, `gryosound.md`, `stocktrend.md`.

Plus `archives.md` which mounts the PaperMod `archives` layout at `/archives/`.

Frontmatter pattern (observed):
```yaml
---
author: "Sean Leishman"
title: "..."
date: "YYYY-MM-DD"
tags: ["Project", "..."]
Summary: "..."
ShowBreadCrumbs: true
ShowToc: false
hideMeta: false
---
```

### Theme (`themes/PaperMod`)
Pulled as a git submodule from `adityatelange/hugo-PaperMod`. Treated as read-only; all customisation lives in the project root.

### Layout overrides (`layouts/`)
Hugo merges these on top of the theme.
- `partials/extend_head.html` — defers jQuery, preconnects Google Fonts, loads IBM Plex Mono and Nanum Gothic Coding, conditionally calls `math.html`.
- `partials/extend_footer.html` — IntersectionObserver + jQuery animation that strokes an SVG `#signature` element when it scrolls into view.
- `partials/math.html` — three KaTeX bundles from jsDelivr (note: two different KaTeX versions are loaded, 0.11.1 and 0.16.4 — see notes.md).
- `partials/post_meta.html` — overrides PaperMod's meta line; only shows the date unless `ShowReadingTime` / `ShowWordCount` are set.
- `shortcodes/newtabref.html` — `{{< newtabref href="..." title="..." >}}` renders an `<a target="_blank" rel="noopener">`.

### Styling (`assets/`)
PaperMod auto-bundles anything under `assets/css/extended/`. Here that's `custom.css` (site-specific overrides; ~4 KB).

### Static (`static/`)
Copied verbatim into `public/`. Holds favicons, avatar, and per-post hero images referenced from `content/projects/*.md` (e.g. `chess.png`, `gyroArch.png`, `stocktrend.png`).

### Build artefacts
- `public/` — Hugo output. Gitignored, but historically a `Build` commit suggests it was once committed. Don't edit.
- `resources/_gen/` — Hugo's image/asset cache.
- `.hugo_build.lock` — empty, used by Hugo during builds.

## Deploy pipeline

Three workflows live in `.github/workflows/`. Effective behaviour:

1. **`hugo.yml`** — runs on push to `master`. Checks out with `submodules: recursive`, installs Hugo (via `peaceiris/actions-hugo`, pinned to `0.64.0` in the step despite an unused `HUGO_VERSION: 0.114.0` env), installs Dart Sass, builds with `hugo --minify --baseURL ""`, uploads `./public` as a Pages artifact, and then a separate `deploy` job calls `FirebaseExtended/action-hosting-deploy@v0` against project `portolfio-2c85b`. Note: the artifact upload is a leftover from a Pages setup; the actual deploy is to Firebase.
2. **`firebase-hosting-merge.yml`** — auto-generated by `firebase init hosting:github`. Also runs on push to `master`, runs `hugo`, deploys to channel `live` on `portolfio-2c85b`.
3. **`firebase-hosting-pull-request.yml`** — auto-generated, runs on PRs. Has a syntactically broken step (two `uses:` keys on the deploy step) — it likely fails or is ignored.

`firebase.json` declares `site: sean-blog`, `public: public`, ignores dotfiles and `node_modules`.

## Data flow at request time
Static — there is no request-time logic. The browser fetches HTML/CSS/JS from Firebase, then KaTeX (jsDelivr) and Google Fonts load from their CDNs, and the signature SVG animation runs client-side once it scrolls into view.
